# E2E：书号 3953 — apibi 路径与 HTML 目录路径对照

**日期:** 2026-04-24  
**范围:** Phase 13 / E2E-01、E2E-02；**不**含全书正文下载。  
**原始采数:** 同目录 `13-RUN-RECORD.md`（可逐命令复现）。  

## 背景

`novel_downloader` 在能连上 apibi 书库且目录接口成功时，走 **apibi 章链**（`apibi://` 伪链 + JSON 拉正文）；否则走 **传统 HTML 目录 + HTTP 拉章页**。本实验对**同一归一化书号** `3953` 分别构造两种可观测路径，并记录**目录级**差异。  

**与 999.1 的关系:** `999.1-apibi-watermark-bom/E2E-2026-04-23.md` 以**水印/BOM 抽测**为主；本文件聚焦 **apibi vs HTML 双路径的目录/元数据差异**，**不**重复 999.1 的摘段。  

## 复现环境

- Python 3、仓库根、`requirements` 已装  
- **路径 A:** 不设置 `BQUGE_API_BASE`（或设为可达的默认 `https://apibi.cc/api`）  
- **路径 B:** `export BQUGE_API_BASE='http://127.0.0.1:1/api'`（使 apibi 书元拉取失败，**强制**无 API 目录，从而走 HTML；见 `13-CONTEXT` D-02）

## 结论与注意

1. **同一书号**下，**apibi 目录**与 **HTML 目录页**解析出的**章数与标题**可以**不一致**；维护与对照时应写明「本次走的是 API 还是 HTML」，不宜只看书号。  
2. 差异**不一定**是客户端 bug，更可能是**站点/索引与 apibi 书库**不同步；结论带**时间戳**，日后可能变化。  
3. 若需「强制」观察 HTML 路径，可采用 `13-RUN-RECORD` 中的 **`BQUGE_API_BASE` 指到本机不可达** 的写法；**不要**将无效 URL 当长期配置写入仓库。
